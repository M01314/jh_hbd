# 모바일 최적화 및 스와이프 조작 구현 계획

모바일 사용자가 QR 코드를 통해 접속했을 때 원활하게 플레이할 수 있도록 조작 방식과 레이아웃을 개선합니다.

## 1. 조작 방식 개선 (JavaScript)
- **터치 스와이프 구현**: `touchstart`와 `touchend` 이벤트를 사용하여 사용자의 스와이프 방향을 감지합니다.
  - **위로 스와이프**: 점프 (`handleJump` 실행)
  - **아래로 스와이프**: 수그리기 (`handleDuck(true)` 후 일정 시간 뒤 또는 `touchend` 시 `handleDuck(false)`)
- **기존 터치 클릭**: 기본적으로 점프 기능을 유지하되, 스와이프와 충돌하지 않도록 조정합니다.

## 2. 레이아웃 최적화 (CSS)
- **미디어 쿼리 추가**: `@media (max-width: 600px)`를 사용하여 모바일 환경에 맞는 스타일을 적용합니다.
- **게임 영역 조정**: 
  - `#game-wrapper`의 높이를 화면 높이에 맞춰 조정하거나 비율을 조정합니다.
  - 캐릭터(`player`, `pursuer-girl`)의 위치를 픽셀 대신 유연한 단위로 변경합니다.
- **UI 요소 크기 조절**: 
  - 제목, 버튼, 미니맵 등의 크기를 모바일 화면에 최적화합니다.
  - 편지 카드(`message-card`)의 패딩과 폰트 크기를 줄여 가독성을 높입니다.

## 3. 테스트 및 마무리
- 모바일 브라우저 환경에서 스와이프 감도가 적절한지 확인합니다.
- 화면 회전(가로/세로 모드) 시 레이아웃이 깨지지 않는지 점검합니다.
